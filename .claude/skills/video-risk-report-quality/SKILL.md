# 视频风险报告质量审核 Skill

## 目标

本 Skill 用于保证 video-risk-analyzer 输出的 Word 报告更接近人工巡检/运营交付标准。

重点不是增加风险点数量，而是提升报告质量：
- 风险类型清晰
- 不重复堆砌普通风险
- 强风险不漏
- 描述谨慎可信
- 长期/临时风险判断合理
- 方便运营、测试、交付人员复核

## 风险类型标准化

报告中不要出现"其他、其他"。

优先使用以下标准类型：
- 施工 / 修路
- 施工围挡
- 锥桶 / 临时导流
- 限高 / 净空核查 / 低净空
- 桥洞 / 隧道 / 顶棚
- 闸口 / 门岗 / 护栏 / 隔离墩
- 窄路 / 通行空间受限
- 非机动车混行
- 行人横穿
- 停车占道
- 商铺门口 / 出入口密集
- 大型车辆遮挡
- 货车 / 工程车占道
- 视线遮挡
- 路面异常
- 物流装卸区
- 其他待复核

如果确实无法判断，最多使用"其他待复核"，不要输出"其他、其他"。

## 重复风险控制

普通风险不要连续重复输出。

普通风险包括：
- 非机动车
- 普通停车占道
- 树木遮挡
- 普通商铺门口
- 普通视线遮挡
- 动态车辆干扰

规则：
- 同一路段非机动车混行，只保留 1-2 个代表性点。
- 同一路段停车占道，只保留 1-2 个代表性点。
- 树木遮挡通常最多保留 1 个，除非遮挡交通标志、限高牌、闸口、路口视线。
- 商铺门口停车占道不要连续输出多个，只保留典型点。
- 同类普通风险 60-120 秒内不要重复输出。
- 不要为了凑数量输出弱风险。

## 强风险保留

以下风险必须优先保留：
- 施工 / 修路
- 施工围挡
- 锥桶 / 临时导流
- 固定闸口 / 门岗 / 隔离墩
- 限高 / 低净空 / 桥洞 / 顶棚
- 大型车辆明显遮挡
- 货车 / 工程车占道
- 通行空间明显收窄
- 路面异常
- 明显需要人工复核或低速通过的点

强风险不能因为普通类别上限被删除。

## 相邻风险合并

如果两个风险点相隔 10-20 秒以内，并且属于同一连续场景，应优先合并。

例如：
施工围挡 + 闸口收窄 + 大型车辆遮挡  
应合并为一个综合风险点。

合并描述示例：
"该路段存在施工围挡、闸口收窄及大型车辆遮挡，通行空间受限，建议低速通过并人工复核。"

## 限高 / 净空判断

没有明确限高牌或限高数值时，不要直接写"限高"。

应写：
- 净空核查
- 疑似低净空
- 隧道 / 顶棚结构
- 建议人工复核通行高度

只有出现明确限高牌、限高数值或明显低净空结构时，才写"限高"。

## 长期/临时风险判断

每个风险点建议包含风险属性：

risk_attribute:
- 长期/阶段性风险
- 临时/动态风险
- 待复核

长期/阶段性风险：
- 修路
- 施工围挡
- 固定闸口
- 固定门岗
- 固定隔离设施
- 固定低净空
- 桥洞 / 隧道 / 顶棚
- 长期道路封闭
- 军事基地、园区管制出入口

临时/动态风险：
- 普通非机动车
- 行人
- 临时停车
- 临时货车
- 临时遮挡
- 普通商铺门口车辆

待复核：
- 看不清是否长期存在的停车占道
- 疑似固定障碍
- 疑似低净空但无明确标识

停车占道不要轻易判断为长期风险。
只有明显固定占道、同一位置多次出现，或结合场景判断可能一周以上不变化，才写"建议长期关注"。

## 描述可信度

报告描述要谨慎，不要过度绝对。

避免使用：
- 一定
- 必然
- 压缩约1/4
- 路面有碎石
- 需绕行
- 净空高度较低

除非画面非常清楚。

推荐使用：
- 画面可见
- 疑似
- 可能
- 建议低速通过
- 建议人工复核
- 当前画面无法确认是否长期存在

## 风险点数量原则

风险点数量不是固定值。

原则：
- 风险少：可以 3-5 个
- 风险正常：通常 5-10 个
- 风险较多：可以 10-12 个
- 不要为了凑数量输出弱风险
- 不要重复堆砌普通风险
- 强风险不能漏
- 普通风险只保留代表性样例

目标是：数量适中、代表性强、方便人工复核。

## Word 报告检查清单

生成报告前检查：

1. 是否还有"其他、其他"
2. 是否存在 10-20 秒内同场景重复点
3. 普通非机动车是否过多
4. 普通停车占道是否过多
5. 树木遮挡是否过多
6. 限高/净空措辞是否谨慎
7. 是否区分长期/临时/待复核
8. 描述是否有过度绝对表述
9. 强风险是否保留
10. 最终数量是否适中
