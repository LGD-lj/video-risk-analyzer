# 视频风险识别调参 Skill

## 调参目标

让系统接近人工/GPT 巡检水平。
目标不是固定输出 5-10 个，而是根据实际视频输出数量适中、代表性强、对运营路线有实际意义的风险点。

## 判断问题类型

### 候选风险点很少

如果候选风险点数量很少，说明：
- 视觉 prompt 太保守
- 抽帧太少
- 风险阈值太高
- 模型只识别事故风险，没有识别运营风险

优先调整：
- 降低 MIN_RISK_SCORE
- 开启 RISK_RECALL_MODE
- 强化 prompt：运营风险也算风险
- 增加抽帧密度

### 候选很多，最终很少

如果候选很多但最终输出很少，说明：
- 去重过狠
- MIN_GAP_SECONDS 过大
- DeepSeek 最终筛选删太多
- 类别上限过低

优先调整：
- 降低强风险去重间隔
- 区分强风险和普通风险去重
- 让 DeepSeek 只合并重复，不要删除代表性风险

### 输出很多但重复

如果最终输出很多但重复，说明：
- 类别上限不足
- 普通风险去重不足
- 相似描述聚类不足

优先调整：
- 非机动车最多 1-2 个
- 树木遮挡最多 1 个
- 停车占道最多 2 个
- 商铺门口停车只保留 1-2 个典型点
- 普通风险去重间隔 60-90 秒

### 强风险漏检

如果施工、限高、闸口、锥桶漏检：
- 强化视觉 prompt
- 强风险 risk_score 下限保持较低
- 强风险不要被普通类别上限误删
- 抽帧间隔适当缩短

## 推荐配置

正式分析模式：
- ANALYSIS_MODE=full
- RISK_RECALL_MODE=true
- FRAME_INTERVAL_SECONDS=3 或 5
- MIN_RISK_SCORE=40
- MIN_GAP_SECONDS=30
- MAX_RISK_POINTS=12

快速测试模式：
- ANALYSIS_MODE=quick
- FRAME_INTERVAL_SECONDS=15
- MAX_ANALYZE_FRAMES=40

## 评估指标

每次测试后必须汇报：
- 视频时长
- 抽帧数量
- 视觉模型分析帧数
- 原始候选风险点数量
- 按类型统计的候选数量
- 去重后数量
- 最终输出数量
- 各类型最终保留数量
- Word 是否生成
- ZIP 是否生成
- 原始视频和临时帧是否清理

## 调参原则

一次只改一类参数：
- 要么改 prompt
- 要么改抽帧
- 要么改阈值
- 要么改去重
- 不要一次改太多，避免不知道是哪项生效
